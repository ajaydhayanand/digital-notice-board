2. Final Tech Stack and Why

Frontend: React 18 Styling: TailwindCSS HTTP: Axios Backend: Node.js + Express Database: MySQL (mysql2) Auth: JWT + bcryptjs Security: helmet + validation

Reasoning: React gives reusable component architecture and fast UI development; Express provides lightweight REST API design; MySQL offers relational consistency for users/notices/read states; JWT is suitable for stateless role-based authorization.

3. High-Level Architecture and Flow

[React Client] | (Axios + JWT Authorization header) v [Express API Layer] |-- /api/auth/login |-- /api/notices (CRUD + filters + read/important) |-- /api/notices/stream (SSE for live notice change events) v [MySQL Database] |-- users |-- notices |-- notice_reads Live Update Path: Admin action -> DB write -> EventEmitter emit -> SSE pushes event -> clients refresh list Fallback path: periodic polling every few seconds

4. Database Schema (Implemented)

TablePurposeMain Fields
usersStores login accounts and rolesid, username, password_hash, role(admin/student), created_at
noticesStores all published noticesid, title, description, category, created_by, is_important, created_at
notice_readsTracks per-user read state for each noticeid, user_id, notice_id, read_at (unique user_id+notice_id)

Indexes exist for performance on category, title, created_at, important flag, and read-join keys.

12. Common Viva Questions (Quick Answers)

  1. Why MySQL instead of MongoDB? SRS and relational read-state tracking fit better with joins and constraints.
  2. How is RBAC enforced? JWT carries role; backend requireAdmin blocks non-admin writes.
  3. How do students and admin coexist simultaneously? Per-tab session with sessionStorage; each tab keeps independent auth state.
  4. How are live updates handled? SSE push from backend event bus, with fallback polling.
  5. How is read/unread stored? Separate notice_reads table with unique user-notice pair.
  6. How is important notice feature implemented? is_important column + admin-only PATCH route + highlighted UI.
  7. How are inputs validated? Using express-validator on route payloads and params.
  8. How is API secured? JWT middleware, role middleware, secure headers, hashed passwords, controlled CORS.
  9. How is pagination done? SQL count + limit/offset + pagination metadata in API response.
  10. What can be future enhancements? file attachments, notice expiry, email/push notifications, audit logs, unit/integration tests, CI/CD pipelines.

13. Demo Script for Viva (2-3 minutes)

  1. Login as admin, show Admin Dashboard.
  2. Create a notice and mark it important.
  3. Edit the notice and save.
  4. Open another tab as student, show notice appears, mark read/unread.
  5. Apply search, category filter, and pagination.
  6. Delete notice as admin and show immediate sync in student view.
